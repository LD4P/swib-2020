RDF Templates

Using RDF to Create RDF

The flexibility of RDF to represent a "meta" layer for describing resources in Sinopia means that catalogers use the same user interface for both RDF templates and cataloging the resources with those templates.

Below is a simple example of an RDF template created for a Schema.org Book.

Schema.org Book RDF Template Sample

        <> <http://sinopia.io/vocabulary/hasResourceTemplate> "sinopia:template:resource";
            a <http://sinopia.io/vocabulary/ResourceTemplate>;
            <http://sinopia.io/vocabulary/hasClass> <http://schema.org/Book>;
            <http://sinopia.io/vocabulary/hasPropertyTemplate> _:b1.
        _:b1 <http://www.w3.org/1999/02/22-rdf-syntax-ns#rest> <http://www.w3.org/1999/02/22-rdf-syntax-ns#nil>;
            <http://www.w3.org/1999/02/22-rdf-syntax-ns#first> _:b2.
        _:b2 a <http://sinopia.io/vocabulary/PropertyTemplate>;
            <http://sinopia.io/vocabulary/hasPropertyUri> <http://schema.org/name>;
            <http://www.w3.org/2000/01/rdf-schema#label> "A Name"@eng.

JSON Resource Templates

Sinopia initially used a JSON Domain Specific Language (DSL) developed for use by the Library of Congress in their original BIBFRAME Editor.

As Sinopia matured, more functionality was desired and the Sinopia team explored and then adopted an RDF template vocabuary at https://sinopia.io/vocabulary that allows catalogers to manage templates within the editor as any RDF resource.
