SWIB 2020 - Sinopia Linked Data Editor

Sinopia's Application Programming Interface (API)

Migrating from Trellis

In the first two iterations of Sinopia, all of the RDF and JSON data was stored and managed through the Linked-Data Platform Trellis running on Amazon Web Services cloud with their Relational Database Service (RDS) to store and manage the data.

As Sinopia matured, the scability of Trellis was a concern as the trival amount of RDF and JSON being managed was hitting performance bottlenecks. Other issues with Trellis was the extremely small user base and the inflexibility of Trellis to manage other data required by requested new features for Sinopia.

OpenAPI

The Sinopia API is described with an OpenAPI available at https://ld4p.github.io/sinopia_api/.

A significant change to the Sinopia stack was the addition of a separate API service, running on AWS, that offers CRUD management of RDF templates and resources within the editor.

Moving away from Trellis, the Sinopia development team switch to using DocumentDB, AWS's cloud hosted version of MongoDB. Moving to MongoDB addressed the scability concerns as well as provided a more flexible data storage for the new personalization features being implemented in Sinopia.

The Sinopia API also offers easier access for external uses like individual institutional triplestores and machine learning opportunities like the RDF Classifier.
