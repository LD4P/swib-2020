SWIB 2020 - Sinopia Linked Data Editor

Background

Funded by the Andrew W. Mellon Foundation, the Sinopia Editor and back-end supporting services is currently on version 3.5.6 and is developed primarily by Stanford University Libraries.

Workcycle One 2018-2019

Initially a fork of the first Library of Congress BIBFRAME Editor code base, the Sinopia Editor was heavily refactored using the front-end Javascript React framework composed of web-based components using Redux for state managment.

Back-end RDF management was accomplished using Trellis, a Linked-Data Platform supported by a Elasticsearch search index.

The Sinopia Stack is deployed on Amazon Web Services (AWS) to better support a collaborative cataloging environment for multiple institutions.

Workcycle Two 2019

First production use of Sinopia to create RDF descriptions by 17 cohort institutions. Many of the institutions were involved with different original cataloging projects that used Sinopia in their work.

User interface improvements were the main focus for this work-cycle and establishing work-flows for JSON-based Resource Templates.

Workcycle Three 2020

Sinopia migrated off of Trellis in favor of AWS DocumentDB, major change from using JSON Resource Templates to using RDF-based templates and depreciating the dedicated Profile Editor application.

Created an API providing CRUD operations along with user customization and BIBFRAME specific enhancements.

Current Architecture

The Sinopia technology stack has evolved over the three work-cycles but at the broadest level is a client-side React single page application (SPA) connecting to a server-side API running as Node.js applications on Amazon Web Services (AWS). There are three server-side helper applications that index incoming templates and resources, export resources, and offer a BIBFRAME RDF-to-MARC conversion.
